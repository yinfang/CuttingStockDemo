# 一维下料问题求解器开发文档

## 项目概述

一维下料问题求解器是一个使用遗传算法解决一维下料优化问题的WPF应用程序。该应用程序可以帮助用户优化材料使用，减少浪费，提高生产效率。

## 项目结构

```
CuttingStockProblem/
├── src/                         # 主项目源代码
│   ├── Models/                  # 数据模型
│   ├── Algorithms/              # 算法实现
│   ├── Services/                # 服务类
│   ├── ViewModels/              # 视图模型
│   ├── Views/                   # 视图
│   ├── Commands/                # 命令模式实现
│   ├── App.xaml                 # 应用程序入口
│   └── CuttingStockProblem.csproj # 项目文件
├── tests/                       # 测试项目
│   ├── GeneticAlgorithmTests.cs # 算法测试
│   └── CuttingStockProblem.Tests.csproj # 测试项目文件
└── docs/                        # 文档
```

## 功能特性

### 主要功能
1. **遗传算法优化**：使用遗传算法解决一维下料问题
2. **数据输入**：支持输入原材料和切割需求
3. **参数配置**：可配置遗传算法参数
4. **结果展示**：清晰展示优化结果
5. **Excel导出**：支持将结果导出到Excel

### 界面功能
1. **数据输入界面**：
   - 原材料管理（长度、数量、成本）
   - 切割项目管理（长度、需求数量）

2. **算法配置界面**：
   - 种群大小设置
   - 最大迭代次数设置
   - 交叉率设置
   - 变异率设置
   - 损耗值设置

3. **结果展示界面**：
   - 解决方案摘要（总模式数、总浪费、效率、总成本）
   - 详细结果展示
   - 导出功能

## 安装和运行

### 系统要求
- .NET 6.0 SDK 或更高版本
- Windows 7 或更高版本

### 安装步骤
1. 克隆或下载项目源代码
2. 确保已安装 .NET 6.0 SDK
3. 在项目根目录下运行以下命令：

```bash
# 恢复项目依赖
dotnet restore

# 编译项目
dotnet build

# 运行项目
dotnet run --project src/CuttingStockProblem.csproj
```

### 测试
运行测试套件：
```bash
dotnet test
```

## 核心组件

### 数据模型

#### StockMaterial（原材料）
表示可用的原材料
- Length: 原材料长度
- Quantity: 原材料数量
- Cost: 原材料成本

#### CuttingItem（切割项目）
表示需要切割的项目
- Length: 切割长度
- Quantity: 需求数量
- Demand: 需求次数

#### CuttingPattern（切割模式）
表示一个切割模式
- Stock: 使用的原材料
- CutItems: 切割的项目列表
- Waste: 浪费量

#### SolutionResult（解决方案）
表示优化结果
- Patterns: 切割模式列表
- TotalWaste: 总浪费量
- TotalStocksUsed: 使用的原材料总数
- TotalCost: 总成本
- Efficiency: 效率百分比

### 算法实现

#### GeneticAlgorithm（遗传算法）
实现遗传算法来解决一维下料问题：

1. **初始化**：创建随机种群
2. **适应度评估**：计算每个个体的适应度
3. **选择**：使用轮盘赌选择机制
4. **交叉**：一点交叉操作
5. **变异**：随机交换变异
6. **新种群创建**：精英保留策略

### 服务类

#### ExcelExportService（Excel导出服务）
提供将结果导出到Excel文件的功能

### 界面组件

#### MainWindow（主窗口）
主应用程序窗口，包含三个标签页：
1. 数据输入标签页
2. 算法设置标签页
3. 结果展示标签页

#### App.xaml
应用程序入口点和全局资源定义

## 开发指南

### 添加新功能

1. **创建新模型**：
   - 在Models文件夹中创建新的模型类
   - 确保实现适当的接口和继承关系

2. **添加新算法**：
   - 在Algorithms文件夹中创建新的算法类
   - 实现相应的接口

3. **添加新服务**：
   - 在Services文件夹中创建新的服务类
   - 注册到依赖注入容器中

### 界面开发

1. **添加新视图**：
   - 在Views文件夹中创建新的XAML文件
   - 创建对应的视图模型

2. **本地化**：
   - 所有界面文本都应支持多语言
   - 使用资源文件管理本地化字符串

### 测试

1. **单元测试**：
   - 为所有算法和业务逻辑编写单元测试
   - 使用xUnit测试框架

2. **集成测试**：
   - 测试组件之间的集成

## 故障排除

1. **编译错误**：

2. **运行时错误**：

   - 避免内存泄漏
